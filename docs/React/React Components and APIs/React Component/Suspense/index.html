<Suspense>

Syntax​

<Suspense fallback={<Loading />}>
  <SomeComponent />
</Suspense>
  • children : 你想要渲染的 UI。
  • fallback : 一個在等待 children 加載時顯示的 React 元素。

Usage​

Displaying a fallback while content is loading​

只有兩種情況會顯示 fallback :

  1. 在 Suspense-enabled frameworks (Next.js ...) 執行 Data-fetching
  2. 使用 lazy 的組件

Suspense 不會等待 Effect 或者 Event Handlers ，所以你需要自己處理。

<Suspense fallback={<Loading />}>
  <Albums />
</Suspense>

Revealing content together at once​

會等待所有組件都跑完才顯示

<Suspense fallback={<Loading />}>
  <Biography />
  <Panel>
    <Albums />
  </Panel>
</Suspense>

Revealing nested content as it loads​

<Suspense fallback={<BigSpinner />}>
  <Biography />
  <Suspense fallback={<AlbumsGlimmer />}>
    <Panel>
      <Albums />
    </Panel>
  </Suspense>
</Suspense>

Showing stale content while fresh content is loading​

使用 useDeferredValue 來傳遞舊的版本，直到新的版本準備好。

import { Suspense, useState, useDeferredValue } from "react";
import SearchResults from "./SearchResults.js";

export default function App() {
  const [query, setQuery] = useState("");
  const deferredQuery = useDeferredValue(query);
  const isStale = query !== deferredQuery;
  return (
    <>
      <label>
        Search albums:
        <input value={query} onChange={(e) => setQuery(e.target.value)} />
      </label>
      <Suspense fallback={<h2>Loading...</h2>}>
        <div style={{ opacity: isStale ? 0.5 : 1 }}>
          <SearchResults query={deferredQuery} />
        </div>
      </Suspense>
    </>
  );
}

Preventing already revealed content from hiding​

當一個組件被 Suspense 時，他的父組件也會被 Suspense ，這可能造成畫面的不和諧，這時候你可能需要 startTransition 來避免這種情況。

Indicating that a transition is happening​

你可以使用 useTransition` 表示在 loading 的狀態

Resetting Suspense boundaries on navigation​

如果你要前往一個帶有參數的路徑，你可能會需要用 key 告訴 React 他們是不同的東西。

Providing a fallback for server errors and server-only content​

如果你使用 streaming server rendering APIs 的話，你可以使用 <Suspense> 來處理可能發生的錯誤。
