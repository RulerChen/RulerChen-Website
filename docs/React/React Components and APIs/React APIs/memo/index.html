memo

Syntax​

import { memo } from "react";

const SomeComponent = memo(function SomeComponent(props) {
  // ...
});

Usage​

Skipping re-rendering when props are unchanged​

如果傳入的 props 不改變就不會重新渲染。

Updating a memoized component using state​

如果傳入的 props 不改變，但是 state 改變，那麼也會重新渲染。

Updating a memoized component using a context​

如果傳入的 props 不改變，但是 context 改變，那麼也會重新渲染。

Minimizing props changes​

在 javascript 中，{} !== {}，所以即使傳入的函數一樣也會重新渲染，因此我們可以使用 useCallback、useMemo 來避免這個問題， 或者是你可以盡量減少你的 props。

Specifying a custom comparison function​

在很少的情況下你可以在memo中自訂比較函數，但是通常不需要這麼做，在使用的時候也要確定這樣會比 re-render 更快。

const Chart = memo(function Chart({ dataPoints }) {
  // ...
}, arePropsEqual);

function arePropsEqual(oldProps, newProps) {
  return (
    oldProps.dataPoints.length === newProps.dataPoints.length &&
    oldProps.dataPoints.every((oldPoint, index) => {
      const newPoint = newProps.dataPoints[index];
      return oldPoint.x === newPoint.x && oldPoint.y === newPoint.y;
    })
  );
}
