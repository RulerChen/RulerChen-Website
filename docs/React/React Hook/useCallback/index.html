useCallback

Syntax​

const testFunction = useCallback(() => {
  fn();
}, [dependencies]);

Parameters​

  • fn : 你想要鎖住的函數
  • dependencies : 甚麼時候要發生變化

Return​

在第一次渲染時，會回傳 fn function ，如果 dependencies 沒有發生變化，則會回傳同一個 fn function。

Usage​

Skipping re-rendering of components​

在預設的情況下，如果有一個 component 被重新渲染， React 就會重新渲染他所有的 child component ，造成效能浪費。

在 javascript 中， function 每次都會被重新創建( ref 會不一樣)，所以如果你把 function 傳給 child component ，那麼 child component 就會被重新渲染，memo就會變得沒有意義，所以我們可以使用 useCallback 來鎖住 function ，讓他不會被重新創建。

如這份程式碼所示，當你點擊 +1 按鈕時， child component 不會被重新渲染。

  • App.jsx
  • ChildComponent.jsx

如這份程式碼所示，當你點擊 +1 按鈕時， child component 不會被重新渲染。

import { useState, useCallback } from "react";

import ChildComponent from "./ChildComponent";

export default function Page() {
  const [count, setCount] = useState(0);
  console.log("main render");

  const handleSubmit = useCallback(() => {
    console.log("submit");
  }, []);

  return (
    <div>
      <button onClick={() => setCount((c) => c + 1)}>+1</button>
      <p>{count}</p>
      <ChildComponent onSubmit={handleSubmit} />
    </div>
  );
}

如這份程式碼所示，當你點擊 +1 按鈕時， child component 不會被重新渲染。

import React, { memo } from "react";

const ChildComponent = memo(function ChildComponent({ onSubmit }) {
  console.log("ChildComponent render");
  return (
    <div>
      <button onClick={onSubmit}>Submit</button>
    </div>
  );
});

export default ChildComponent;

Updating state from a memoized callback​

function TodoList() {
  const [todos, setTodos] = useState([]);

  const handleAddTodo = useCallback((text) => {
    const newTodo = { id: nextId++, text };
    setTodos((todos) => [...todos, newTodo]);
  }, []);
}

Preventing an Effect from firing too often​

在下面的例子中，如果我們不使用 useCallback ，則每次 render 時都會重新創建 createOptions ，導致 useEffect 也會重新執行，造成效能浪費。

function ChatRoom({ roomId }) {
  const [message, setMessage] = useState("");

  const createOptions = useCallback(() => {
    return {
      serverUrl: "https://localhost:1234",
      roomId: roomId,
    };
  }, [roomId]); // ✅ Only changes when roomId changes

  useEffect(() => {
    const options = createOptions();
    const connection = createConnection();
    connection.connect();
    return () => connection.disconnect();
  }, [createOptions]); // ✅ Only changes when createOptions changes
}

Optimizing a custom Hook​

如果使用 custom hook 的話，可以使用 useCallback 來鎖住 function ，避免讓其他相關的 component 被重新創建。

function useRouter() {
  const { dispatch } = useContext(RouterStateContext);

  const navigate = useCallback(
    (url) => {
      dispatch({ type: "navigate", url });
    },
    [dispatch]
  );

  const goBack = useCallback(() => {
    dispatch({ type: "back" });
  }, [dispatch]);

  return {
    navigate,
    goBack,
  };
}

Tips​

useCallback vs useMemo​

useCallback 是用於 cache 你的 function ，而 useMemo 是 cache 你的結果。

Should you add useCallback everywhere?​

useCallback 只有在以下兩種情況有用

  1. 你將這個函數當成 props 傳給被 memo 包住的 component 。
  2. 這個函數被當成其他 Hook 的 dependencies 。

useCallback 並不會防止創建函數也不會不執行函數，但 React 會返還相同的函數參考，除非依賴項發生變化。
