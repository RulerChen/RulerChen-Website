useLayoutEffect

Syntax​

import { useState, useRef, useLayoutEffect } from "react";

function Tooltip() {
  const ref = useRef(null);
  const [tooltipHeight, setTooltipHeight] = useState(0);

  useLayoutEffect(() => {
    const { height } = ref.current.getBoundingClientRect();
    setTooltipHeight(height);
  }, []);
}

Parameters​

  • setUp : 和 useEffect 一樣有 setUp 和 cleanUp ，但是 useLayoutEffect 會在 component 被加到 DOM 之前執行，所以不會發生畫面閃爍的問題。
  • dependencies : 和 useEffect 一樣，可以用來控制 setUp 的執行時機。

Return​

useLayoutEffect 回傳 undefined。

Caveats​

useLayoutEffect 會損失效能，我們應該優先使用 useEffect。

Usage​

Measuring layout before the browser repaints the screen​

大多數的 component 不需要知道他們所在的位置，但我們有時候仍然需要它們的資訊。 比如說我們要做一個 tooltip ，我們需要知道 tooltip 的高度，才能夠決定要在哪裡顯示 tooltip。 為了做到這件事情，我們需要這樣做 :

  1. 隨便找個位置 render 他
  2. 測量它的位置
  3. 再把它 render 到正確的位置 你不會希望使用者可以看到 tooltip 在移動，所以你需要 useLayoutEffect 。
function Tooltip() {
  const ref = useRef(null);
  const [tooltipHeight, setTooltipHeight] = useState(0); // You don't know real height yet

  useLayoutEffect(() => {
    const { height } = ref.current.getBoundingClientRect();
    setTooltipHeight(height); // Re-render now that you know the real height
  }, []);

  // ...use tooltipHeight in the rendering logic below...
}
