useRef

Syntax​

import { useRef } from "react";

function MyComponent() {
  const intervalRef = useRef(0);
  const inputRef = useRef(null);
}

Parameters​

  • initialValue : 初始值

Return​

回傳一個只有一個屬性的物件

  • current : 綁定物件的屬性

Caveats​

  • 與其他 Hook 不同， useRef.current 是 muttable 的，且不會觸發 re-render。
  • 不要在 render 的時候使用 useRef，這會產生不可預測的行為。
function MyComponent() {
  // ...
  // 🚩 Don't write a ref during rendering
  myRef.current = 123;
  // ...
  // 🚩 Don't read a ref during rendering
  return <h1>{myOtherRef.current}</h1>;
}

Usage​

Referencing a value with a ref​

改變 ref 的時候並不會觸發 re-render ，因此 useRef 最適合用來處理不會改動畫面的東西。

在使用 useRef 時需要注意以下幾件事：

  1. 你可以儲存 re-render 之間的訊息
  2. 不會觸發 re-render
  3. useRef 儲存特定組件的資訊，而不會共享給其他組件
import { useState, useRef } from "react";

export default function Stopwatch() {
  const [startTime, setStartTime] = useState(null);
  const [now, setNow] = useState(null);
  const intervalRef = useRef(null);

  function handleStart() {
    setStartTime(Date.now());
    setNow(Date.now());

    clearInterval(intervalRef.current);
    intervalRef.current = setInterval(() => {
      setNow(Date.now());
    }, 10);
  }

  function handleStop() {
    clearInterval(intervalRef.current);
  }

  let secondsPassed = 0;
  if (startTime != null && now != null) {
    secondsPassed = (now - startTime) / 1000;
  }

  return (
    <>
      <h1>Time passed: {secondsPassed.toFixed(3)}</h1>
      <button onClick={handleStart}>Start</button>
      <button onClick={handleStop}>Stop</button>
    </>
  );
}

Manipulating the DOM with a ref​

使用 useRef 直接操作 DOM 元素

import { useRef } from "react";

export default function CatFriends() {
  const listRef = useRef(null);

  function scrollToIndex(index) {
    const listNode = listRef.current;
    const imgNode = listNode.querySelectorAll("li > img")[index];
    imgNode.scrollIntoView({
      behavior: "smooth",
      block: "nearest",
      inline: "center",
    });
  }

  return (
    <>
      <nav>
        <button onClick={() => scrollToIndex(0)}>Tom</button>
        <button onClick={() => scrollToIndex(1)}>Maru</button>
        <button onClick={() => scrollToIndex(2)}>Jellylorum</button>
      </nav>
      <div>
        <ul ref={listRef}>
          <li>
            <img src="https://placekitten.com/g/200/200" alt="Tom" />
          </li>
          <li>
            <img src="https://placekitten.com/g/300/200" alt="Maru" />
          </li>
          <li>
            <img src="https://placekitten.com/g/250/200" alt="Jellylorum" />
          </li>
        </ul>
      </div>
    </>
  );
}

Avoiding recreating the ref contents​

React 只會在第一次 render 時建立 ref 的內容，之後的 render 都會使用同一個 ref ，所以不要使用以下的方法：

function Video() {
  const playerRef = useRef(new VideoPlayer());
}

應該使用以下的方法

function Video() {
  const playerRef = useRef(null);

  function getPlayer() {
    if (playerRef.current !== null) {
      return playerRef.current;
    }
    const player = new VideoPlayer();
    playerRef.current = player;
    return player;
  }
}

Tips​

I can't get a ref to a custom component​

const inputRef = useRef(null);

return <MyInput ref={inputRef} />;

你可以把你的 MyInput 包在 forwardRef 裡面，這樣就可以使用 ref 了。

import { forwardRef } from "react";

const MyInput = forwardRef(({ value, onChange }, ref) => {
  return <input value={value} onChange={onChange} ref={ref} />;
});

export default MyInput;
