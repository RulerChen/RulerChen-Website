useEffect

Syntax​

useEffect(() => {
  setUp();
  return () => {
    cleanUp();
  };
}, [dependencies]);

Parameters​

  • setup : 當 component 被加到 DOM 上時，執行的 function ，每一次 re-render 時，會先執行 cleanUp function 再執行 setup function。
  • dependencies :

Return​

useEffect 回傳 undefined。

Caveats​

  • 在 strictMode 下， React 會先執行一次 development-only setup+cleanup cycle ，以確保 setUp 和 cleanUp 的對應關係。

Usage​

Connecting to an external system​

import { useEffect } from "react";
import { createConnection } from "./chat.js";

function ChatRoom({ roomId }) {
  const [serverUrl, setServerUrl] = useState("https://localhost:1234");

  useEffect(() => {
    const connection = createConnection(serverUrl, roomId);
    connection.connect();
    return () => {
      connection.disconnect();
    };
  }, [serverUrl, roomId]);
  // ...
}

可以將 useEffect 包裝在 custom hook 中，以便重複使用。

Controlling a non-React widget​

使外部系統與 React 保持一致。

Fetching data with Effects​

原則上鼓勵使用 React 框架或者是依些開源套件，比如 react-query。

在下方的程式碼中，我們使用 ignore 來避免 race condition (或者你也可以使用 AbortController)。

import { useState, useEffect } from "react";
import { fetchBio } from "./api.js";

export default function Page() {
  const [person, setPerson] = useState("Alice");
  const [bio, setBio] = useState(null);

  useEffect(() => {
    let ignore = false;
    setBio(null);
    fetchBio(person).then((result) => {
      if (!ignore) {
        setBio(result);
      }
    });
    return () => {
      ignore = true;
    };
  }, [person]);
}

Specifying reactive dependencies​

把所有可互動的變數都放在 dependencies 中，以確保 useEffect 會在這些變數改變時被觸發。

可互動的變數(Reactive values)包括 props 和定義在 component 中的 變數和函數。

警告

不要使用 eslint-ignore-next-line react-hooks/exhaustive-deps 來忽略 dependencies 的檢查。

Updating state based on previous state from an Effect​

import { useState, useEffect } from "react";

export default function Counter() {
  const [count, setCount] = useState(0);

  useEffect(() => {
    const intervalId = setInterval(() => {
      setCount((c) => c + 1); // ✅ Pass a state updater
    }, 1000);
    return () => clearInterval(intervalId);
  }, []); // ✅ Now count is not a dependency

  return <h1>{count}</h1>;
}

Removing unnecessary object and function dependencies​

如果有 object or function 作為 dependencies ，每一次 re-render 時，都會重新建立一個新的 object or function ，這樣會導致 useEffect 每一次都會被觸發，如果可以的話盡量移到 useEffect 裡面。

const serverUrl = "https://localhost:1234";

function ChatRoom({ roomId }) {
  const [message, setMessage] = useState("");

  // 🚩 This object is created from scratch on every re-render
  const options = {
    serverUrl: serverUrl,
    roomId: roomId,
  };

  useEffect(() => {
    const connection = createConnection(options); // It's used inside the Effect
    connection.connect();
    return () => connection.disconnect();
  }, [options]); // 🚩 As a result, these dependencies are always different on a re-render
}

Reading the latest props and state from an Effect​

function Page({ url, shoppingCart }) {
  const onVisit = useEffectEvent((visitedUrl) => {
    logVisit(visitedUrl, shoppingCart.length);
  });

  useEffect(() => {
    onVisit(url);
  }, [url]); // ✅ All dependencies declared
}

Displaying different content on the server and the client​

如果你的程式碼需要在 server 和 client 上有不同的行為，可以使用 useEffect 來達成。

function MyComponent() {
  const [didMount, setDidMount] = useState(false);

  useEffect(() => {
    setDidMount(true);
  }, []);

  if (didMount) {
    // ... return client-only JSX ...
  } else {
    // ... return initial JSX ...
  }
}

Tips​

My Effect does something visual, and I see a flicker before it runs​

如果你的 useEffect 會做一些視覺上的變化，你可能會看到一些閃爍的情況，這是因為 useEffect 會在 re-render 之後才會被觸發， 如果你想要避免這種情況，可以使用 useLayoutEffect 來達成。
