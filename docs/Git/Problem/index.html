Git 常見問題

我的 git 設定​

  1. git config --global user.name "<your_name>" : 設定名字
  2. git config --global user.email "<your_email>" : 設定 email
  3. git config --global core.autocrlf input : 設定換行符號
  4. git config --global init.defaultBranch main : 設定預設 branch 名稱為 main

如何推送到 GitHub​

  1. git init : 初始化一個 git repository
  2. git add . : 將所有檔案加入 git 暫存區
  3. git commit -m "<commit_message>" : 將暫存區的檔案 commit 到本地端
  4. git remote add origin <GitHub repository url> : 將本地端的 repository 與 GitHub repository 連結
  5. git push -u origin <your_branch_name> : 將本地端的 repository 推送到 GitHub repository，<your_branch_name> 可能是 main 或 master。

如何修改 commit message​

以下有四種方式 :

  1. 使用 git commit -m --amend '<commit_message>' 來修改最後一次的 commit 紀錄
  2. 使用 git reset --hard HEAD^ 來刪除最後一次的 commit 紀錄
  3. 使用 git rebase -i HEAD^ 來修改多個 commit 紀錄
    將你要改的 commit 紀錄由 pick 改成 r (reword)，儲存之後再修改 commit 紀錄。
    vim 的操作方法為按下 i 進入編輯模式，修改完之後按下 esc 鍵，再輸入 :wq 儲存並離開。
  4. 開大絕 => 刪除 .git 檔案夾，重新初始化 git repository

reset 的三種模式​

  1. git reset --soft <commit_hash> : 工作區跟暫存區都不會變
  2. git reset --mixed <commit_hash> : (default) 工作區不變，暫存區丟掉
  3. git reset --hard <commit_hash> : 工作區和暫存區都會被丟掉

找回被 reset 的 commit 紀錄​

  1. 使用 git reflog 或 git log -g 找到被 reset 的 commit 紀錄
  2. 使用 git reset --hard <commit hash> 回到 commit 紀錄

如何使用標籤​

  1. git tag <tag_name> <commit_hash> : 使用輕量標籤
  2. git tag -am "<tag_desc>" <tag_name> : 使用標示標籤

如何合併分支​

  1. 使用 git merge <branch_name> 來合併兩條分支並產生一個新的節點
  2. 使用 git rebase <branch_name> 來把原本的分支複製到目前的分支上

如何取消 rebase​

以下兩種方式 :

  1. 使用 reflog 查詢並使用 git reset --hard <commit_hash> 回到 rebase 前的狀態
  2. git reset ORIG_HEAD --hard : 使用 ORIG_HEAD 來回到 rebase 前的狀態

放下手邊的工作去做其他事情​

有以下兩種方法 :

  1. 先 commit 手上的東西，等修完之後再使用 git reset 繼續目前的工作
  2. 使用 git stash
    1. git stash -u : 暫存目前的工作狀態 (有 Untrack 檔案需加 -u)
    2. git stash list : 查看 stash 的狀態
    3. git stash pop "<stash_id>" : 把東西用在目前的分支上，並刪除 stash
    4. git stash drop "<stash_id>" : 刪除 stash
    5. git stash apply "<stash_id>" : 把東西用在目前的分支上，但不刪除 stash

不小心 commit 到帳號密碼怎麼辦​

  1. git filter-branch -f --tree-filter "rm -f <file_path>" : 只能清除 commit，但仍然可以恢復，你需要額外處理資源回收的問題
  2. rm .git/refs/original/refs/heads/master : 處理 .git 的備份點
  3. git reflog expire --all --expire=now : 處理 reflog
  4. git gc --prune=now : 處理資源回收
  5. git push -f : 強制推送到遠端

如何進行 commit 的合併或刪除​

  1. git rebase -i <commit_hash> : 進入互動模式
  2. 合併 : 使用 squash 進行合併
  3. 刪除 : 使用 drop 進行刪除
  4. 新增 : 使用 edit 進行新增
