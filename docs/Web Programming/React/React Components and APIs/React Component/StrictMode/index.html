<StrictMode>

Syntax​

import { StrictMode } from "react";
import { createRoot } from "react-dom/client";

const root = createRoot(document.getElementById("root"));
root.render(
  <StrictMode>
    <App />
  </StrictMode>
);
  • 你的組件會額外 re-render 來檢查由 impure function 所產生的 bug。
  • 你的組件會額外執行一次 Effect 去檢查 cleanup function 是否正確執行。
  • 檢查是否有廢棄的 APIs。
  • 只會在 development mode 下執行。

Usage​

Enabling Strict Mode for entire app​

官方推薦在 root component 使用 StrictMode。

Enabling strict mode for a part of the app​

import { StrictMode } from "react";

function App() {
  return (
    <>
      <Header />
      <StrictMode>
        <main>
          <Sidebar />
          <Content />
        </main>
      </StrictMode>
      <Footer />
    </>
  );
}

Fixing bugs found by double rendering in development​

React 假定每個組件都是 pure function ，strict mode 會對一些 function 多執行一次，包括 :

  1. component function body (並不包含 event handler 內部)
  2. 你傳給 useState, set functions, useMemo, or useReducer 的函數
  3. 一些 class component 如 constructor, render, and shouldComponentUpdate

Fixing bugs found by re-running Effects in development​

執行兩次可以幫助你找出一些 side effects 造成的 bug。

Fixing deprecation warnings enabled by Strict Mode​

幫助找出一些已經廢棄的 API。
