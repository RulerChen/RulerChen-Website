攔截路由

習慣​

路由攔截可以使用 (..) 的方式進行定義，這類似於相對路徑慣例 ../ ，但是針對路由段落。

你可以使用以下方式：

  • (.) 用於匹配同一級別的路由段落
  • (..) 用於匹配上一級別的路由段落
  • (..)(..) 用於匹配上兩級別的路由段落
  • (...) 用於匹配從根 app 目錄開始的路由段落

例如，你可以通過創建一個 (..)photo 目錄，在 feed 段落內攔截 photo 段落。

例子​

攔截路由可以與平行路由一起使用，以創建視窗。

使用這種模式來創建模態視窗可以克服一些常見的問題，包括：

  • 通過 URL 分享視窗的內容
  • 在刷新頁面時保留上下文，而不是關閉視窗
  • 在後退導向時關閉視窗，而不是返回上一個路由
  • 在前進導向時重新打開視窗
