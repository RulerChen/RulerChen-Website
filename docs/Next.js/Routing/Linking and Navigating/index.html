連結和導向

<Link> 組件​

<Link>是一個 React 組件，擴展了 HTML 的<a>元素，提供了預取和在客戶端之間進行路由導航的功能。它是在 Next.js 中進行路由導航的主要方法。

import Link from "next/link";

export default function Page() {
  return <Link href="/dashboard">Dashboard</Link>;
}

範例​

連接到動態路由​

import Link from "next/link";

export default function PostList({ posts }) {
  return (
    <ul>
      {posts.map((post) => (
        <li key={post.id}>
          <Link href={`/blog/${post.slug}`}>{post.title}</Link>
        </li>
      ))}
    </ul>
  );
}

檢查活動連結​

你可以使用 usePathname 來檢查當前的路由。

"use client"

import { usePathname } from "next/navigation";
import Link from "next/link";

export function Navigation({ navLinks }) {
  const pathname = usePathname();

  return (
    <>
      {navLinks.map((link) => {
        const isActive = pathname.startsWith(link.href);

        return (
          <Link className={isActive ? "text-blue" : "text-black"} href={link.href} key={link.name}>
            {link.name}
          </Link>
        );
      })}
    </>
  );
}

滑動到id​

<Link> 默認是到頁面的頂部，你也可以提供 id 讓他滑到對應的位置。

useRouter Hook​

useRouter 鉤子允許您在客戶端組件內更改路由。

"use client"

import { useRouter } from "next/navigation";

export default function Page() {
  const router = useRouter();

  return (
    <button type="button" onClick={() => router.push("/dashboard")}>
      Dashboard
    </button>
  );
}

useRouter 提供了 push() 和 refresh() 等等的方法，但是除非必要，最好使用 <Link> 組件。

頁面導向是如何運作的​

  • 我們可以使用 <Link> 或 Router.push() 來使頁面導向。
  • 路由器會更新瀏覽器的 URL
  • 接著路由器會從客戶端緩存中使用未更改的段落，這也叫做部分渲染。
  • 如果滿足軟導向的條件，路由器將從緩存中獲取新的部分。如果不滿足條件，路由器將執行硬導向並從服務器獲取服務器組件。
  • 在載入期間，將顯示載入的 UI，而路由器會使用緩存或新的資料來渲染新的部分。
  • 如果路由器從服務器獲取了新的部分，則會將其添加到客戶端緩存中。

渲染服務器組件的客戶端緩存​

新的路由器具有一個在記憶體中的客戶端緩存，用於存儲已渲染的服務器組件的結果。 該緩存以路由段落為單位，允許在任何層級進行失效，並確保在並發渲染中的一致性。

當用戶在應用程序中導向時，路由器會將先前獲取的段落和預取的段落存儲在緩存中。

緩存失效​

服務器操作可通過路徑（revalidatePath）或緩存標籤（revalidateTag）重新驗證數據。

預取​

預取是一種在訪問之前在後台預先加載路由的方式。 預取的路由渲染結果會被添加到路由器的客戶端緩存中。 這樣，當訪問預取的路由時，可以實現幾乎即時的響應。

默認情況下，使用 <Link> 組件時，當路由在視窗中可見時，它們會在首次加載頁面或通過滾動時進行預取。 也可以使用 useRouter() 鉤子的 prefetch 方法預取路由。

靜態路由和動態路由：

如果路由是靜態的，則會預取該路由段落的所有服務器組件載荷。 如果路由是動態的，則會預取從第一個共享佈局到第一個 loading.js 檔案之間的組件載荷。這樣可以減少動態預取整個路由的成本，並為動態路由實現即時的加載狀態。

預取只有在生產模式下才會發生，我們也可以使用 <Link prefetch={false}> 來避免預取。

軟導向​

在導向過程中，對於已更改的路由段落，Next.js 會使用緩存（如果存在），並且不會向服務器發出新的數據請求。

軟導向的條件 :
在導向過程中，如果您要訪問的路由已被預取，並且不包含動態路由段落和與當前路由相同的動態參數，則 Next.js 將使用軟導向。

例如，考慮包含動態 [team] 路由段落的路由：/dashboard/[team]/。 在 /dashboard/[team]/ 下的緩存路由段落只有在 [team] 參數更改時才會失效。

  • 從 /dashboard/team-red/_ 導向到 /dashboard/team-red/_ 將是一個軟導向。
  • 從 /dashboard/team-red/_ 導向到 /dashboard/team-blue/_ 將是一個硬導向。

硬導向​

在導向過程中，緩存將被失效，服務器將重新獲取數據並重新渲染已更改的路由段落。

返回/前進導向​

返回和前進導向（popstate 事件）是軟導航行為。 客戶端緩存將被重用，導向幾乎是即時的。

焦點和滾動​

默認情況下，Next.js 將在導向時設置焦點並將滾動定位到已更改的路由段落。
