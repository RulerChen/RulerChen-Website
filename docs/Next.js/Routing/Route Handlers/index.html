路由處理程序

習慣​

路由處理程序（Route Handlers）定義在 app 目錄內的 route.js 或 route.ts 文件中：

支持的 HTTP 方法 支持以下 HTTP 方法：GET、POST、PUT、PATCH、DELETE、HEAD 和 OPTIONS。 如果使用了不支持的方法，Next.js 將返回 405 Method Not Allowed 響應。

擴展的 NextRequest 和 NextResponse API 除了支持原生的 Request 和 Response 對象外，Next.js 通過擴展它們的功能，提供了 NextRequest 和 NextResponse，以便提供方便的輔助功能。

行為​

靜態路由處理​

app/items/route.ts
import { NextResponse } from "next/server";

export async function GET() {
  const res = await fetch("https://data.mongodb-api.com/...", {
    headers: {
      "Content-Type": "application/json",
      "API-Key": process.env.DATA_API_KEY,
    },
  });
  const data = await res.json();

  return NextResponse.json({ data });
}

動態路由處理​

在以下情況下，路由處理程序將以動態方式處理：

  • 使用 Request 物件和 GET 方法。
  • 使用任何其他 HTTP 方法。
  • 使用動態函數，例如 cookies 和 headers。
  • 手動指定配置選項中的動態模式。
app/products/api/route.ts
import { NextResponse } from "next/server";

export async function GET(request: Request) {
  const { searchParams } = new URL(request.url);
  const id = searchParams.get("id");
  const res = await fetch(`https://data.mongodb-api.com/product/${id}`, {
    headers: {
      "Content-Type": "application/json",
      "API-Key": process.env.DATA_API_KEY,
    },
  });
  const product = await res.json();

  return NextResponse.json({ product });
}

同樣地，使用 POST 方法將導致路由處理程序以動態方式進行評估。

app/items/route.ts
import { NextResponse } from "next/server";

export async function POST() {
  const res = await fetch("https://data.mongodb-api.com/...", {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      "API-Key": process.env.DATA_API_KEY,
    },
    body: JSON.stringify({ time: new Date().toISOString() }),
  });

  const data = await res.json();

  return NextResponse.json(data);
}

例子​

重新驗證靜態資料​

app/items/route.ts
import { NextResponse } from "next/server";

export async function GET() {
  const res = await fetch("https://data.mongodb-api.com/...", {
    next: { revalidate: 60 }, // Revalidate every 60 seconds
  });
  const data = await res.json();

  return NextResponse.json(data);
}

動態函式​

路由處理程序可以與 Next.js 的動態函數一起使用，例如 cookies 和 headers。

可以使用 next/headers 模組中的 cookies 函數來讀取 cookie。 此伺服器函數可以直接在路由處理程序中調用，或者嵌套在其他函數內部。 該 cookies 是只讀的。 要設置 cookie，您需要返回一個包含 Set-Cookie 標頭的新 Response。

app/api/route.ts
import { cookies } from "next/headers";

export async function GET(request: Request) {
  const cookieStore = cookies();
  const token = cookieStore.get("token");

  return new Response("Hello, Next.js!", {
    status: 200,
    headers: { "Set-Cookie": `token=${token.value}` },
  });
}

可以使用 next/headers 模組中的 headers 函數來讀取 headers。 此伺服器函數可以直接在路由處理程序中調用，或者嵌套在其他函數內部。 該 headers 是只讀的。要設置標頭，您需要返回一個包含新標頭的新 Response。

app/api/route.ts
import { headers } from "next/headers";

export async function GET(request: Request) {
  const headersList = headers();
  const referer = headersList.get("referer");

  return new Response("Hello, Next.js!", {
    status: 200,
    headers: { referer: referer },
  });
}

重新導向​

app/api/route.ts
import { redirect } from "next/navigation";

export async function GET(request: Request) {
  redirect("https://nextjs.org/");
}

動態路由段落​

app/items/[slug]/route.ts
export async function GET(request: Request, { params }: { params: { slug: string } }) {
  const slug = params.slug; // 'a', 'b', or 'c'
}

串流​

串流是常與大型語言模型（LLM）結合使用的，例如 OpenAI，用於生成 AI 內容。

app/api/completion/route.ts
import { Configuration, OpenAIApi } from "openai-edge";
import { OpenAIStream, StreamingTextResponse } from "ai";

const config = new Configuration({
  apiKey: process.env.OPENAI_API_KEY,
});
const openai = new OpenAIApi(config);

export const runtime = "edge";

export async function POST(req: Request) {
  const { prompt } = await req.json();
  const response = await openai.createCompletion({
    model: "text-davinci-003",
    stream: true,
    temperature: 0.6,
    prompt: "What is Next.js?",
  });

  const stream = OpenAIStream(response);
  return new StreamingTextResponse(stream);
}

請求的主體​

app/items/route.ts
import { NextResponse } from "next/server";

export async function POST(request: Request) {
  const res = await request.json();
  return NextResponse.json({ res });
}

CORS​

app/api/route.ts
export async function GET(request: Request) {
  return new Response("Hello, Next.js!", {
    status: 200,
    headers: {
      "Access-Control-Allow-Origin": "*",
      "Access-Control-Allow-Methods": "GET, POST, PUT, DELETE, OPTIONS",
      "Access-Control-Allow-Headers": "Content-Type, Authorization",
    },
  });
}

Edge 和 Node.js 環境​

Route Handlers 具有同樣的 API，可無縫支援 Edge 和 Node.js 執行時環境，包括對流式傳輸的支援。 由於 Route Handlers 使用與 Pages 和 Layouts 相同的路由段配置，因此它們支援一些期待已久的功能，例如通用的靜態再生成的 Route Handlers。

沒有 UI 的回應​

您可以使用 Route Handlers 返回非 UI 內容。請注意， sitemap.xml、robots.txt、應用程式圖示和 Open Graph 圖片都具有內建支援。

app/rss.xml/route.ts
export async function GET() {
  return new Response(`<?xml version="1.0" encoding="UTF-8" ?>
<rss version="2.0">
 
<channel>
  <title>Next.js Documentation</title>
  <link>https://nextjs.org/docs</link>
  <description>The React Framework for the Web</description>
</channel>
 
</rss>`);
}
