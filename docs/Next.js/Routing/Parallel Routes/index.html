平行路由

慣例​

並行路由允許你有條件地渲染插槽（slot），從而在相同的 URL 上實現完全分離的程式碼。

未匹配的路由​

默認情況下，插槽中渲染的內容將與當前的 URL 匹配。 在未匹配的插槽情況下，Next.js 根據路由技術和文件結構的不同渲染不同的內容。

你可以定義一個 default.js 文件，當 Next.js 無法根據當前的 URL 恢復插槽的活動狀態時，作為後備內容進行渲染。

如果你從根目錄 / 導航到 /settings，根據導航的類型和 default.js 文件的可用性，渲染的內容將有所不同。

with @analytics/default.jswithout @analytics/default.js
Soft Navigation@team/settings/page.js and @analytics/page.js@team/settings/page.js and @analytics/page.js
Hard Navigation@team/settings/page.js and @analytics/default.js404 Error
  • 軟導向 : 即使插槽的活動狀態與當前的 URL 不匹配，Next.js 也將渲染插槽的先前活動狀態。
  • 硬導向 : 需要完整頁面重新加載的導航 - Next.js 將首先嘗試渲染未匹配插槽的 default.js 文件。如果不存在 default.js 文件，則會渲染 404 頁面。

useSelectedLayoutSegment(s)​

useSelectedLayoutSegment 和 useSelectedLayoutSegments 都接受 parallelRoutesKey 參數，它允許你在該插槽內讀取活動的路由段。

app/layout.tsx
'use client'
 
import { useSelectedLayoutSegment } from 'next/navigation'
 
export default async function Layout(props: {
  //...
  authModal: React.ReactNode
}) {
  const loginSegments = useSelectedLayoutSegment('authModal')
  // ...
}

當使用者導向到 @authModal/login 或在網址列輸入 /login 時，loginSegments 的值將為 "login"。

例子​

平行路由可用於渲染視窗。

app/layout.tsx
export default async function Layout(props: {
  // ...
  authModal: React.ReactNode;
}) {
  return (
    <>
      {/* ... */}
      {props.authModal}
    </>
  );
}
app/@authModal/login/page.tsx
import { Modal } from "components/modal";

export default function Login() {
  return (
    <Modal>
      <h1>Login</h1>
      {/* ... */}
    </Modal>
  );
}
app/@authModal/login/default.tsx
export default function Default() {
  return null;
}

如果你要關閉視窗，你可以將使用 <Link> 或 router.back()

app/@authModal/login/page.tsx
"use client";
import { useRouter } from "next/navigation";
import { Modal } from "components/modal";

export default async function Login() {
  const router = useRouter();
  return (
    <Modal>
      <span onClick={() => router.back()}>Close modal</span>
      <h1>Login</h1>
      ...
    </Modal>
  );
}

你也可以使用 Catch-All 路由。

條件路由​

你可以從登入狀態決定要渲染的內容。

app/layout.tsx
import { getUser } from "@/lib/auth";

export default function Layout({ dashboard, login }: { dashboard: React.ReactNode; login: React.ReactNode }) {
  const isLoggedIn = getUser();
  return isLoggedIn ? dashboard : login;
}
