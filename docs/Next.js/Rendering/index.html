渲染

渲染環境​

客戶端是指用戶設備上的瀏覽器，它向服務器發送請求以獲取應用程序代碼。然後它將服務器的響應轉換為用戶可以交互的界面。

服務器是指資料中心中的電腦，用於存儲程式碼、接收來自客戶端的請求、執行一些計算並發回適當的回應。

組件的客戶端和服務器渲染​

在 React 18 之前，使用 React 渲染應用程序的主要方式完全是在客戶端上。

Next.js 提供了一種更簡單的方法來將應用程序分解為頁面並通過生成 HTML 並將其發送到客戶端進行水化來在服務器上進行預渲染通過反應。然而，這導致客戶端需要額外的 JavaScript 來使初始 HTML 具有交互性。

現在，有了服務端和客戶端組件，React 可以在客戶端和服務器上渲染，這意味著您可以在組件級別選擇渲染環境。

默認情況下，app 路由器使用 Server Components，允許您輕鬆地在服務器上渲染組件並減少發送到客戶端的 JavaScript 量。

服務器中靜態和動態的渲染​

通過靜態渲染，服務器和客戶端組件都可以在構建時在服務器上預渲染。工作結果被緩存並在後續請求中重用。緩存的結果也可以重新驗證。

靜態渲染​

服務器和客戶端組件在靜態渲染期間的渲染方式不同：

客戶端組件的 HTML 和 JSON 已預渲染並緩存在服務器上。然後將緩存的結果發送到客戶端進行水合作用。 服務器組件由 React 在服務器上渲染，它們的有效負載用於生成 HTML。相同的渲染有效負載還用於混合客戶端上的組件，從而導致客戶端不需要 JavaScript。

動態渲染​

通過動態渲染，服務器和客戶端組件都在請求時在服務器上渲染。工作結果不會被緩存。

Edge 和 Node.js 執行環境​

在服務器上，有兩個運行時可以渲染頁面：

Node.js 運行時（默認）可以訪問生態系統中的所有 Node.js API 和兼容包。 Edge Runtime 基於 Web API。 兩個運行時都支持來自服務器的流式傳輸，具體取決於您的部署基礎架構。
