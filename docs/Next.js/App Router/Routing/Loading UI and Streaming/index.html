載入 UI 與分流

即時載入狀態​

即時載入狀態是在導向時立即顯示的後備用戶界面。 您可以預先渲染載入指示器，例如骨架屏和旋轉圖示，或者是未來畫面中的一個小但有意義的部分，例如封面照片、標題等。 這有助於用戶理解應用正在回應並提供更好的用戶體驗。

使用 Suspense 進行分流​

除了 loading.js 外，您還可以手動為自己的 UI 組件創建 Suspense。App Router 支持在 Node.js 和 Edge 環境下使用 Suspense 進行串流。

什麼是分流？​

在 SSR 中，需要完成一系列步驟才能讓用戶看到並與頁面交互：

  1. 所有頁面的數據在伺服器端進行獲取。
  2. 伺服器將渲染頁面的 HTML。
  3. 將頁面的 HTML、CSS 和 JavaScript 發送到客戶端。
  4. 使用生成的 HTML 和 CSS 顯示非交互式用戶界面。
  5. React 對用戶界面進行 hydration ，使其變得可交互。

這些步驟是按照順序進行的，且是阻塞的，這意味著伺服器只能在所有數據獲取完成後才能渲染頁面的 HTML。 同樣，在客戶端上，React 只能在頁面中的所有組件代碼下載完成後才能 hydrate 用戶界面。

使用 React 和 Next.js 的 SSR 可以改善感知的加載性能，通過盡快向用戶顯示非交互式頁面。 然而，它仍然可能很慢，因為在頁面顯示給用戶之前，需要完成伺服器上的所有數據獲取。

分流允許將頁面的 HTML 拆分為更小的塊，並逐步將這些塊從伺服器發送到客戶端。 這使得頁面的部分內容可以更早地顯示，而無需等待所有數據加載完成才能渲染任何用戶界面。

分流與 React 的組件模型非常契合，因為每個組件都可以被視為一個塊。 具有更高優先級的組件（例如產品信息）或不依賴於數據的組件（例如佈局）可以優先發送，React 可以更早地開始 hydrate。 具有較低優先級的組件（例如評論、相關產品）可以在其數據獲取完成後通過同一個伺服器請求發送。

分流在以下情況下特別有益：當您想要防止長時間數據請求阻塞頁面的渲染時，它可以減少首字節時間（TTFB）和首次有內容渲染時間（FCP）。 它還有助於提高互動時間（TTI），尤其是在較慢的設備上。

例子​

<Suspense> 通過將執行異步操作的組件（例如數據獲取）包裹起來，顯示在操作進行時的後備用戶界面（例如骨架屏、旋轉圖示），然後在操作完成後切換為您的組件。

使用 <Suspense>，您可以獲得以下好處：

  • 串流伺服器渲染（Streaming Server Rendering）- 逐步從伺服器向客戶端渲染 HTML。
  • 選擇性補水（Selective Hydration）- React 根據用戶交互來優先確定哪些組件首先變為可交互。

SEO​

在生成元數據（generateMetadata）中，Next.js 將等待數據獲取完成後再將 UI 進行串流到客戶端。 這樣可以保證串流響應的第一部分包含了 <head> 標籤。
