專案組織與檔案放置

默認情況下的安全共置​

在 app 目錄中，嵌套的資料夾結構定義了路由結構。

儘管路由結構是通過資料夾來定義的，但在將 page.js 或 route.js 文件添加到路由之前，該路由將無法公開訪問。

即使一個路由被公開訪問，也只有由 page.js 或 route.js 返回的內容被發送給客戶端。 這意味著項目文件可以安全地共存於 app 目錄中的路由段內。

專案結構特點​

私人資料夾​

我們可以用_來創建私人資料夾

在默認情況下，app 目錄中的檔案可以安全地共置，因此不需要私有資料夾進行共置。 然而，私有資料夾在以下情況下可能會有用：

  • 將 UI 邏輯與路由邏輯分開。
  • 在整個項目和 Next.js 生態系統中一致地組織內部文件。
  • 在文字編輯器中對檔案進行排序和分組。
  • 避免與未來 Next.js 文件規範可能的命名衝突。

群組路由​

路由群組對於以下方面非常有用：

  • 將路由按照站點部分、意圖或團隊等進行分組。
  • 在同一路由段級別上啟用嵌套佈局：
    • 在同一段落中創建多個嵌套佈局，包括多個根佈局。
    • 將佈局添加到常見段中的子路由子集中。

src 目錄中​

Next.js 可以將程式碼放在 src 目錄中。

模組路徑別名​

// before
import { Button } from "../../../components/button";

// after
import { Button } from "@/components/button";
