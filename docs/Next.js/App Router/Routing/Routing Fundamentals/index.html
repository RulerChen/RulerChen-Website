概述

路由是每個應用程式的基礎，這個章節會介紹 Next.js 中基本的路由觀念。

術語​

  • 樹狀結構 (Tree) ：一種用於視覺化的結構。例如，一個具有父組件和子組件的組件樹，一個資料夾結構等等。
  • 子樹 (Subtree) ：子樹是樹的一部分，從新的根節點（第一個）開始，到葉節點（最後一個）結束。
  • 根節點 (root) ：樹或子樹中的第一個節點，例如根佈局（root layout）。
  • 葉節點 (leaf) ：子樹中沒有子節點的節點，例如 URL 路徑中的最後一個段落。

  • URL 段落 (URL Segment) ：由/分隔出的 URL 路徑的一部分。
  • URL 路徑 (URL Path) ：出現在域名之後的 URL 的一部分，由多個段落組成。

The app Router​

在 13 版中，Next.js 引入了一個基於 React Server Components 的新版 App Router，它支援共享佈局、巢狀路由、加載狀態、錯誤處理等功能。

App Router 在一個名為app的目錄下。
app 可以和 pages 目錄同時存在，但是 app router 的優先級比較高，如果有重覆的路徑會在 built-time 時報錯。

在預設情況下，在app資料夾下的組件會是 React 的服務器組件，但你也可以使用客戶端組件。

資料夾和檔案的角色​

Next.js 使用基於檔案系統的路由器，其中資料夾用來定義路由，而 page.js 檔案用來顯示內容。

路由段落​

每個資料夾都代表一部分 URL 路徑，資料夾名稱就是路徑段落的名稱。

巢狀路由​

資料夾可以嵌套，這樣就可以創建巢狀路由。

檔案名稱​

  • layout : 用於一個路由和他的子路由的佈局
  • page : 用於一個路由的內容
  • loading : 用於一個路由和他的子路由加載 UI 時
  • not-found : 用於一個路由和他的子路由的未找到 UI 時
  • error : 用於一個路由和他的子路由段錯誤時的 UI
  • global-error : 全局錯誤的 UI
  • route : 服務器端的 API 端點
  • template : 特定重新渲染佈局的 UI
  • default : 平行路由的替補 UI

共置文件​

除了特殊名稱的檔案外，你也可以將組件等等的檔案放在 app 目錄下。

使用客戶端導向的服務器中心路由​

與使用客戶端路由的pages目錄不同，App Router 使用服務器中心路由來配合服務器組件和抓取資料。 透過服務器中心路由，客戶端無需下載路由映射的資訊，且同一個服務器組件的請求可以用於查找路由。 這種優化對於所有應用程序都很有用，特別是有許多路由的應用程序。

儘管路由是服務器中心，但路由器依舊使用 <Link> 組件進行客戶端導向，這類似於單頁應用程序(SPA)。 這意味著當用戶導航到新的路由時，瀏覽器不會重新加載頁面，而是更新 URL 並重新渲染有改變的路由段落。

此外，當用戶在應用程序中導向時，路由器將在內存中的客戶端中緩存 React 服務器組件的結果。 緩存以路由段落分割，可以在任何級別進行失效 (假如某部分的數據發生變化)，並確保 React 的並發渲染一致性。 這意味著在某些情況下，可以重用先前提取的路由段落的緩存，進一步提高性能。

部分渲染​

當在同級路由之間進行導航（/test/123、/test/456），Next.js 僅會提取和渲染路由中有變化的佈局和頁面。

進階路由方法​

  • 平行路由（Parallel Routes）: 允許你在同一個路由段落中渲染多個路由。

  • 攔截路由（Intercepting Routes）: 允許您攔截一個路由並將其顯示在另一個路由的上下文中。
