動態路由

慣例​

可以通過在資料夾名稱外加上方括號來創建一個動態段落。如[slug]。

動態段落將作為 params 屬性傳遞給 layout、page、route 和 generateMetadata 函數。

例子​

app/blog/[slug]/page.tsx
export default function Page({ params }: { params: { slug: string } }) {
  return <div>My Post: {params.slug}</div>;
}
RoutesURLParams
app/blog/[slug]/page.js/blog/a{ slug: 'a' }
app/blog/[slug]/page.js/blog/b{ slug: 'b' }
app/blog/[slug]/page.js/blog/c{ slug: 'c' }

創造靜態變數​

generateStaticParams 函數可以與動態路由段結合使用，以在構建時靜態生成路由，而不是在請求時動態生成。

app/blog/[slug]/page.tsx
export async function generateStaticParams() {
  const posts = await fetch("https://.../posts").then((res) => res.json());

  return posts.map((post) => ({
    slug: post.slug,
  }));
}

generateStaticParams 函數的主要好處在於其智能的數據擷取。 如果在 generateStaticParams 函數中使用 fetch 請求來獲取內容，這些請求將自動去除重覆地請求。 這意味著具有相同參數的 fetch 請求在多個 generateStaticParams、佈局和頁面中只會被執行一次，減少構建時間。

通配符路由​

動態路由段落可以通過在方括號內添加...來捕獲所有後續段落，如 [...folderName]。

RoutesURLParams
app/blog/[...slug]/page.js/blog/a{ slug: ['a'] }
app/blog/[...slug]/page.js/blog/a/b{ slug: ['a', 'b'] }
app/blog/[...slug]/page.js/blog/a/b/c{ slug: ['a', 'b', 'c'] }

可選通配符路由​

通配符段落也可以通過在雙方括號中包含參數來設置為可選的，如 [[...folderName]]。

RoutesURLParams
app/blog/[[...slug]]/page.js/blog{}
app/blog/[[...slug]]/page.js/blog/a{ slug: ['a'] }
app/blog/[[...slug]]/page.js/blog/a/b{ slug: ['a', 'b'] }
app/blog/[[...slug]]/page.js/blog/a/b/c{ slug: ['a', 'b', 'c'] }

Typescript​

app/blog/[slug]/page.tsx
export default function Page({ params }: { params: { slug: string } }) {
  return <h1>My Page</h1>;
}
RoutesParams type
app/blog/[slug]/page.js{ slug: string }
app/blog/[...slug]/page.js{ slug: string[] }
app/[categoryId]/[itemId]/page.js{ categoryId: string, itemId: string }
