快取

請求緩存​

fetch()​

預設情況下，所有的 fetch() 請求都會自動快取和去重。 這意味著如果你發出了兩次相同的請求，第二次請求將重複使用第一次請求的結果。

app/page.tsx
async function getComments() {
  const res = await fetch("https://..."); // The result is cached
  return res.json();
}

// This function is called twice, but the result is only fetched once
const comments = await getComments(); // cache MISS

// The second call could be anywhere in your application
const comments = await getComments(); // cache HIT

如果符合以下情況，請求不會被快取：

使用動態方法（例如 next/headers、export const POST）並且 fetch 是一個 POST 請求（或使用 Authorization 或 cookie headers） fetchCache 被配置為預設跳過快取 在單個 fetch 上配置了 revalidate: 0 或 cache: 'no-store' 使用 fetch 進行的請求可以指定 revalidate 選項來控制請求的重新驗證頻率。

app/page.tsx
export default async function Page() {
  // revalidate this data every 10 seconds at most
  const res = await fetch("https://...", { next: { revalidate: 10 } });
  const data = res.json();
  // ...
}

React 的 cache()​

React 允許您使用 cache() 函數來快取並消除重複的請求，將包裝函數調用的結果進行記憶化。當相同的函數使用相同的參數被調用時，將重用緩存的值而不是重新運行該函數。

utils/getUser.ts
import { cache } from "react";

export const getUser = cache(async (id: string) => {
  const user = await db.user.findUnique({ id });
  return user;
});
app/user/[id]/layout.tsx
import { getUser } from "@utils/getUser";

export default async function UserLayout({ params: { id } }) {
  const user = await getUser(id);
  // ...
}
app/user/[id]/posts.tsx
import { getUser } from "@utils/getUser";

export default async function Page({ params: { id } }: { params: { id: string } }) {
  const user = await getUser(id);
  // ...
}

儘管上述示例中 getUser() 函數被調用了兩次，但只會向數據庫發送一次查詢。這是因為 getUser() 函數被包裹在 cache() 中，因此第二次請求可以重用第一次請求的結果。

POST 請求和 cache()​

使用 fetch 進行 POST 請求時，會自動進行去重，除非它們位於 POST Route Handler 內部或在讀取 headers()/cookies()之後。 例如，如果您在上述情況下使用 GraphQL 和 POST 請求，可以使用 cache 來進行請求去重。cache 的參數必須是平面的，只包含基本類型。 深層對象不會進行去重匹配。

utils/getUser.ts
import { cache } from "react";

export const getUser = cache(async (id: string) => {
  const res = await fetch("...", { method: "POST", body: "..." });
  // ...
});

使用 cache() 做預加載​

我們建議在執行數據獲取的實用程序或組件中選擇性地暴露 preload() 導出。

components/User.tsx
import { getUser } from "@utils/getUser";

export const preload = (id: string) => {
  // void evaluates the given expression and returns undefined
  // https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Operators/void
  void getUser(id);
};
export default async function User({ id }: { id: string }) {
  const result = await getUser(id);
  // ...
}
app/user/[id]/page.tsx
import User, { preload } from "@components/User";

export default async function Page({ params: { id } }: { params: { id: string } }) {
  preload(id); // starting loading the user data now
  const condition = await fetchCondition();
  return condition ? <User id={id} /> : null;
}

結合 cache() 、 preload() 和 server-only​

utils/getUser.ts
import { cache } from "react";
import "server-only";

export const preload = (id: string) => {
  void getUser(id);
};

export const getUser = cache(async (id: string) => {
  // ...
});

使用這種方法，您可以急於獲取數據，緩存響應，並確保這種數據獲取僅在服務器上發生。

段級緩存​

段級緩存允許您緩存和重新驗證在路徑段中使用的數據。

此機制允許路徑的不同段控制整個路徑的緩存生存期。 路徑層次結構中的每個 page.tsx 和 layout.tsx 都可以導出 revalidate 值，以設置路徑的重新驗證時間。

app/page.tsx
export const revalidate = 60; // revalidate this segment every 60 seconds
