獲取資料

服務器組件中的 async 和 await​

app/page.tsx
async function getData() {
  const res = await fetch("https://api.example.com/...");
  // The return value is *not* serialized
  // You can return Date, Map, Set, etc.

  // Recommendation: handle errors
  if (!res.ok) {
    // This will activate the closest `error.js` Error Boundary
    throw new Error("Failed to fetch data");
  }

  return res.json();
}

export default async function Page() {
  const data = await getData();

  return <main></main>;
}

在客戶端組件的 use​

use 是一個新的 React 函數，它接受一個 Promise，概念上類似於 await。 use 函數處理由函數返回的 Promise，以與組件、Hooks 和 Suspense 相容的方式。

目前不建議在客戶端組件中將 fetch 函數包裝在 use 中，因為這可能觸發多次重新渲染。 如果您需要在客戶端組件中獲取數據，我們建議使用第三方套件，如 SWR 或 React Query。 這些套件提供了更好的數據緩存和管理機制，能夠更有效地處理數據的請求和更新。

獲取靜態資料​

默認情況下，fetch 函數將自動獲取數據並無限期地緩存。

fetch("https://..."); // cache: 'force-cache' is the default

重新驗證數據 要在定時間隔內重新驗證緩存數據，您可以在 fetch() 函數中使用 next.revalidate 選項來設置資源的緩存生命周期（以秒為單位）。

fetch("https://...", { next: { revalidate: 10 } });

獲取動態資料​

fetch("https://...", { cache: "no-store" });

資料獲取模式​

獲取平行資料​

為了最小化客戶端和伺服器之間的瀑布現象，我們建議使用以下模式獲取數據：

artist/[username]/page.tsx
import { getArtist, getArtistAlbums, type Album } from "./api";

export default async function Page({ params: { username } }: { params: { username: string } }) {
  // Initiate both requests in parallel
  const artistData = getArtist(username);
  const albumData = getArtistAlbums(username);

  // Wait for the artist's promise to resolve first
  const artist = await artistData;

  return (
    <>
      <h1>{artist.name}</h1>
      {/* Send the artist information first,
          and wrap albums in a suspense boundary */}
      <Suspense fallback={<div>Loading...</div>}>
        <Albums promise={albumData} />
      </Suspense>
    </>
  );
}

// Albums Component
async function Albums({ promise }: { promise: Promise<Album[]> }) {
  // Wait for the albums promise to resolve
  const albums = await promise;

  return (
    <ul>
      {albums.map((album) => (
        <li key={album.id}>{album.name}</li>
      ))}
    </ul>
  );
}

通過在 Server Component 中的 await 調用之前開始 fetch 請求，每個請求可以同時開始進行數據獲取。這樣設置組件可以避免水瀑流的問題。

我們可以通過同時發起兩個請求來節省時間，但是在兩個 promise 解析之前，用戶不會看到渲染結果。

為了改善用戶體驗，您可以添加一個 suspense boundary 來拆分渲染工作並盡早顯示部分結果：

artist/[username]/page.tsx
import Albums from "./albums";

async function getArtist(username: string) {
  const res = await fetch(`https://api.example.com/artist/${username}`);
  return res.json();
}

async function getArtistAlbums(username: string) {
  const res = await fetch(`https://api.example.com/artist/${username}/albums`);
  return res.json();
}

export default async function Page({ params: { username } }: { params: { username: string } }) {
  // Initiate both requests in parallel
  const artistData = getArtist(username);
  const albumsData = getArtistAlbums(username);

  // Wait for the promises to resolve
  const [artist, albums] = await Promise.all([artistData, albumsData]);

  return (
    <>
      <h1>{artist.name}</h1>
      <Albums list={albums}></Albums>
    </>
  );
}

有順序的獲取資料​

要按順序獲取數據，您可以直接在需要數據的組件中進行獲取，或者可以等待 fetch 的結果在需要它的組件中進行獲取：

app/artist/page.tsx
async function Playlists({ artistID }: { artistID: string }) {
  // Wait for the playlists
  const playlists = await getArtistPlaylists(artistID);

  return (
    <ul>
      {playlists.map((playlist) => (
        <li key={playlist.id}>{playlist.name}</li>
      ))}
    </ul>
  );
}

export default async function Page({ params: { username } }: { params: { username: string } }) {
  // Wait for the artist
  const artist = await getArtist(username);

  return (
    <>
      <h1>{artist.name}</h1>
      <Suspense fallback={<div>Loading...</div>}>
        <Playlists artistID={artist.id} />
      </Suspense>
    </>
  );
}

在路由中阻塞渲染​

在佈局中獲取數據將阻塞所有在其下方的路由段的渲染，只有在數據加載完成後，這些路由段的渲染才能開始。

在 pages 目錄中，使用服務器端渲染的頁面會顯示瀏覽器的加載指示器，直到 getServerSideProps 完成後，才會渲染該頁面的 React 組件。這可以描述為“全有或全無”的數據獲取方式。要麼你擁有整個頁面的數據，要麼沒有數據。

在 app 目錄中，您有更多的選擇可以探索：

首先，您可以使用 loading.js 在從數據獲取函數中實時流式傳輸結果時，從服務器端顯示即時的加載狀態。 其次，您可以將數據獲取放在組件樹的較低層級，僅對需要數據的部分進行渲染阻塞，例如將數據獲取移至特定組件而不是在根佈局中進行獲取。 在可能的情況下，最好在使用數據的路由段中獲取數據。這還可以讓您僅為正在加載的頁面部分顯示加載狀態，而不是整個頁面。

不使用 fetch() 獲取資料​

在無法直接使用和配置 fetch 請求的情況下，您可能無法直接使用 fetch 進行數據獲取，尤其在使用第三方庫（如 ORM 或數據庫客戶端）的情況下。

在無法使用 fetch 但仍然想要控制布局或頁面的緩存或重新驗證行為的情況下，您可以依賴路由段的默認緩存行為，或使用路由段的緩存配置。

默認緩存行為​

任何不直接使用 fetch 的數據獲取庫都不會影響路由的緩存，其緩存行為將根據路由段的靜態或動態性而定。

如果路由段是靜態的（默認），則請求的輸出將與該段的其餘部分一起被緩存並重新驗證（如果已配置）。如果路由段是動態的，則請求的輸出將不會被緩存，在每次請求時重新獲取該段渲染時的數據。

路由段緩存配置​

作為一種臨時解決方案，直到能夠配置第三方查詢的緩存行為，您可以使用路由段配置來自定義整個路由段的緩存行為。

app/page.tsx
import prisma from "./lib/prisma";

export const revalidate = 3600; // revalidate every hour

async function getPosts() {
  const posts = await prisma.post.findMany();
  return posts;
}

export default async function Page() {
  const posts = await getPosts();
  // ...
}
