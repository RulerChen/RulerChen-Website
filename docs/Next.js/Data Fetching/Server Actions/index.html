服務器操作

慣例​

next.config.js
module.exports = {
  experimental: {
    serverActions: true,
  },
};

創建​

Server Actions 可以在兩個地方定義：

  • 在使用它的組件內部（僅適用於 Server Components）
  • 在單獨的檔案中（Client 和 Server Components），以便重用。 您可以在單個檔案中定義多個 Server Actions。

使用 Server Components​

通過在函數主體頂部使用 "use server" 指令，定義一個異步函數來創建 Server Action。 這個函數應該基於 React Server Components 協議具有可序列化的參數和返回值。

app/server-component.tsx
export default function ServerComponent() {
  async function myAction() {
    "use server";
    // ...
  }
}

使用 Client Components​

如果您在 Client Component 內部使用 Server Action，請在單獨的文件中使用 "use server" 指令來創建您的動作。 然後，將 Server Action 導入到您的 Client Component 中：

app/actions.tsx
"use server";

export async function myAction() {
  // ...
}
app/client-component.tsx
"use client";

import { myAction } from "./actions";

export default function ClientComponent() {
  return (
    <form action={myAction}>
      <button type="submit">Add to Cart</button>
    </form>
  );
}

調用​

  • 使用 action：React 的 action 屬性允許在 <form> 元素上調用 Server Action。
  • 使用 formAction：React 的 formAction 屬性允許處理 <button>、<input type="submit"> 和 <input type="image"> 元素在 <form> 內的情況。
  • 使用 startTransition 的自定義調用：通過使用 startTransition，您可以調用 Server Actions 而不使用 action 或 formAction。這種方法禁用了漸進增強。

action​

您可以使用 React 的 action 屬性在表單元素上調用 Server Action。通過 action 屬性傳遞的 Server Actions 在用戶交互時作為異步副作用進行處理。

app/add-to-cart.js
export default function AddToCart({ productId }) {
  async function addItem(data) {
    "use server";

    const cartId = cookies().get("cartId")?.value;
    await saveToDb({ cartId, data });
  }

  return (
    <form action={addItem}>
      <button type="submit">Add to Cart</button>
    </form>
  );
}

formAction​

您可以使用 formAction 屬性來處理表單元素上的表單操作， 例如按鈕（button）、輸入類型為 "submit" 的元素（input type="submit"）和輸入類型為 "image" 的元素（input type="image"）。 formAction 屬性優先於表單的 action 屬性。

app/form.js
export default function Form() {
  async function handleSubmit() {
    "use server";
    // ...
  }

  async function submitImage() {
    "use server";
    // ...
  }

  return (
    <form action={handleSubmit}>
      <input type="text" name="name" />
      <input type="image" formAction={submitImage} />
      <button type="submit">Submit</button>
    </form>
  );
}

使用 startTransition 的自定義調用​

您還可以使用 useTransition 鉤子提供的 startTransition 方法來調用 Server Actions，而不使用 action 或 formAction。 這在您希望在表單、按鈕或輸入元素以外的場景中使用 Server Actions 時非常有用。

app/components/example-client-component.js
"use client";

import { useTransition } from "react";
import { addItem } from "../actions";

function ExampleClientComponent({ id }) {
  let [isPending, startTransition] = useTransition();

  return <button onClick={() => startTransition(() => addItem(id))}>Add To Cart</button>;
}
app/actions.js
"use server";

export async function addItem(id) {
  await addItemToDb(id);
  // Marks all product pages for revalidating
  revalidatePath("/product/[id]");
}

不使用 startTransition 的自定義調用​

如果您不進行 Server Mutations，您可以直接將函數作為 prop 傳遞，就像其他函數一樣。

app/posts/[id]/page.tsx
import kv from "@vercel/kv";
import LikeButton from "./like-button";

export default function Page({ params }: { params: { id: string } }) {
  async function increment() {
    "use server";
    await kv.incr(`post:id:${params.id}`);
  }

  return <LikeButton increment={increment} />;
}
app/post/[id]/like-button.tsx
"use client";

export default function LikeButton({ increment }: { increment: () => Promise<void> }) {
  return (
    <button
      onClick={async () => {
        await increment();
      }}
    >
      Like
    </button>
  );
}

增強功能​

實驗性的 useOptimistic​

實驗性的 useOptimistic 鉤子提供了一種在應用程序中實現樂觀更新（optimistic updates）的方法。 樂觀更新是一種技術，通過使應用程序看起來更加即時回應，增強了用戶體驗。

當調用 Server Action 時，UI 立即更新以反映預期的結果，而不是等待 Server Action 的響應。

app/thread.js
"use client";

import { experimental_useOptimistic as useOptimistic } from "react";
import { send } from "./actions.js";

export function Thread({ messages }) {
  const [optimisticMessages, addOptimisticMessage] = useOptimistic(messages, (state, newMessage) => [
    ...state,
    { message: newMessage, sending: true },
  ]);
  const formRef = useRef();

  return (
    <div>
      {optimisticMessages.map((m) => (
        <div>
          {m.message}
          {m.sending ? "Sending..." : ""}
        </div>
      ))}
      <form
        action={async (formData) => {
          const message = formData.get("message");
          formRef.current.reset();
          addOptimisticMessage(message);
          await send(message);
        }}
        ref={formRef}
      >
        <input type="text" name="message" />
      </form>
    </div>
  );
}

實驗性的 useFormStatus​

實驗性的 useFormStatus 鉤子可以在表單操作中使用，並提供了 pending 屬性。

app/form.js
"use client";

import { experimental_useFormStatus as useFormStatus } from "react-dom";

function Submit() {
  const { pending } = useFormStatus();

  return (
    <input type="submit" className={pending ? "button-pending" : "button-normal"} disabled={pending}>
      Submit
    </input>
  );
}

漸進式增強​

漸進式增強 (Progressive Enhancement) 允許 <form> 在沒有 JavaScript 或 JavaScript 被停用的情況下正常運作。 這使得使用者可以與表單互動並提交數據，即使表單的 JavaScript 還沒有載入或載入失敗。

無論是 Server Form Actions 還是 Client Form Actions，都支持漸進式增強，並使用其中一種策略：

  • 如果直接將 Server Action 傳遞給 <form>，即使 JavaScript 被停用，表單仍然可以互動。
  • 如果將 Client Action 傳遞給 <form>，表單仍然可以互動，但是動作會被放入佇列，直到表單完成水合。 <form> 使用了選擇性水合 (Selective Hydration)，因此它發生得很快。
app/components/example-client-component.js
"use client";

import { useState } from "react";
import { handleSubmit } from "./actions.js";

export default function ExampleClientComponent({ myAction }) {
  const [input, setInput] = useState();

  return (
    <form action={handleSubmit} onChange={(e) => setInput(e.target.value)}>
      {/* ... */}
    </form>
  );
}

在這兩種情況下，表單在水合之前都可以進行互動。 雖然 Server Actions 的額外好處是不依賴於客戶端的 JavaScript，但您仍然可以在需要時使用 Client Actions 來組合其他行為，而不會犧牲互動性。

尺寸大小​

預設情況下，傳送到 Server Action 的請求主體大小上限為 1MB。 這可以防止大量數據被發送到伺服器，從而消耗大量伺服器資源進行解析。

然而，您可以使用實驗性的 serverActionsBodySizeLimit 選項來配置此限制。 它可以接受以字節為單位的數字或由 bytes 支持的任何字符串格式，例如 1000、'500kb' 或 '3mb'。

next.config.js
module.exports = {
  experimental: {
    serverActions: true,
    serverActionsBodySizeLimit: "2mb",
  },
};

範例​

客戶端組件的用處​

import​

在 Client Components 中無法直接定義 Server Actions，但可以透過引入操作從包含頂層 "use server" 指令的文件中導入它們。 要在 Client Components 中使用 Server Actions，您需要創建一個具有頂層 "use server" 指令的文件，並在該文件中定義您的 Server Actions。 然後，您可以將 Server Actions 引入並在 Client Components 中使用它們。 這使您能夠在 Client Components 中利用 Server Actions 的功能，同時保持服務器端和客戶端代碼的分離。

app/actions.js
"use server";

export async function addItem() {
  // ...
}
app/components/example-client-component.js
"use client";

import { useTransition } from "react";
import { addItem } from "../actions";

function ExampleClientComponent({ id }) {
  let [isPending, startTransition] = useTransition();

  return <button onClick={() => startTransition(() => addItem(id))}>Add To Cart</button>;
}

props​

儘管建議導入 Server Actions，但在某些情況下，您可能希望將一個 Server Action 作為 prop 傳遞給 Client Component。

例如，您可能希望在操作中使用動態生成的值。在這種情況下，將 Server Action 作為 prop 傳遞給 Client Component 可能是一個可行的解決方案。 這樣做可以讓您在 Client Component 中使用 Server Action，同時允許傳遞任何必要的動態數據或配置。 請注意，在此情況下，Client Component 將負責在適當的時機調用傳遞的 Server Action。

app/components/example-server-component.js
import { ExampleClientComponent } from "./components/example-client-component.js";

function ExampleServerComponent({ id }) {
  async function updateItem(data) {
    "use server";
    modifyItem({ id, data });
  }

  return <ExampleClientComponent updateItem={updateItem} />;
}
app/components/example-client-component.js
"use client";

function ExampleClientComponent({ updateItem }) {
  async function action(formData: FormData) {
    await updateItem(formData);
  }

  return (
    <form action={action}>
      <input type="text" name="name" />
      <button type="submit">Update Item</button>
    </form>
  );
}

按需重新驗證​

Server Actions 可以用於按需通過路徑（revalidatePath）或緩存標籤（revalidateTag）重新驗證數據。

import { revalidateTag } from 'next/cache'

async function revalidate() {
  'use server'
  revalidateTag('blog-posts')
}

驗證​

在調用 Server Action 之前，可以對傳遞給 Server Action 的數據進行驗證或清理。 例如，您可以創建一個包裝函數，該函數接收 Server Action 作為參數，並返回一個函數，該函數在數據有效時調用 Server Action。

app/actions.js
"use server";

import { withValidate } from "lib/form-validation";

export const action = withValidate((data) => {
  // ...
});
lib/form-validation.js
export function withValidate(action) {
  return async (formData: FormData) => {
    "use server";

    const isValidData = verifyData(formData);

    if (!isValidData) {
      throw new Error("Invalid input.");
    }

    const data = process(formData);
    return action(data);
  };
}

使用標頭​

您可以在 Server Action 中讀取傳入的請求標頭，例如 cookies 和 headers。通過讀取這些標頭，您可以獲取關於請求的附加信息，例如用戶的身份驗證狀態、請求來源等。

import { cookies } from 'next/headers'

async function addItem(data) {
  'use server'

  const cartId = cookies().get('cartId')?.value

  await saveToDb({ cartId, data })
}
import { cookies } from 'next/headers';

async function create(data) {
  'use server';

  const cart = await createCart():
  cookies().set('cartId', cart.id)
  // or
  cookies().set({
    name: 'cartId',
    value: cart.id,
    httpOnly: true,
    path: '/'
  })
}

詞彙表​

動作​

動作（Actions）是 React 中的一個實驗性功能，允許您以響應用戶交互的方式運行異步代碼。

動作不僅適用於 Next.js 或 React Server Components，但它們在 React 的穩定版本中尚不可用。在使用 Next.js 時，您可以選擇使用 React 的實驗性渠道來使用動作。

動作通過元素的 action 屬性來定義。通常，在構建 HTML 表單時，您將 URL 傳遞給 action 屬性。使用動作，React 現在允許您直接傳遞一個函數。

React 還提供了與動作相關的內置解決方案，用於實現樂觀更新。需要注意的是，新的模式仍在開發中，可能會添加新的 API。

表單動作​

表單動作（Form Actions）集成在 Web 標準的< form> API 中，並支持即插即用的漸進增強和加載狀態。類似於 HTML 原生的 formaction。

服務器函數​

在服務器上運行的函數，但可以在客戶端調用。

服務器動作​

作為動作調用的服務器函數。

通過將它們傳遞給表單元素的 action 屬性，可以使服務器動作進行漸進式增強。在任何客戶端 JavaScript 加載之前，表單就已經可以交互。這意味著不需要 React hydration 即可提交表單。

服務器變更​

修改數據並調用重定向（redirect）、revalidatePath 或 revalidateTag 的服務器動作。
