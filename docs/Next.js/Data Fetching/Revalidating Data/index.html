重新驗證資料

背景驗證​

fetch('https://...', { next: { revalidate: 60 } })
export const revalidate = 60 // revalidate this page every 60 seconds

工作原理

  1. 當對於在構建時靜態渲染的路徑進行請求時，它將首先顯示緩存的數據。
  2. 在初始請求之後的 60 秒內對於該路徑的任何請求也會被緩存並且立即返回。
  3. 在 60 秒窗口過後，下一個請求仍然會顯示緩存的（過期）數據。
  4. Next.js 將在後台觸發數據的重新生成。
  5. 一旦路徑成功生成，Next.js 將使緩存失效並顯示更新的路徑。如果後台重新生成失敗，則舊數據將保持不變。

當對於尚未生成的路徑段進行請求時，Next.js 將在第一個請求時動態渲染該路徑。未來的請求將從緩存中提供靜態路徑段。

按需求驗證​

如果將重新驗證時間設置為 60，則所有訪問者在一分鐘內將看到相同的生成版本的網站。唯一使緩存失效的方法是在一分鐘後有人訪問該頁面。

Next.js 應用程式路由器支援根據路徑或緩存標籤按需重新驗證內容。這使得您可以手動清除特定的 Next.js 緩存。

使用按需重新驗證​

可以通過路徑（revalidatePath）或緩存標籤（revalidateTag）按需重新驗證數據。

例如，以下 fetch 添加了緩存標籤 collection：

app/page.js
export default async function Page() {
  const res = await fetch("https://...", { next: { tags: ["collection"] } });
  const data = await res.json();
  // ...
}

然後，可以在 Route Handler 中調用 revalidateTag 來按需重新驗證該緩存數據。

app/api/revalidate/route.ts
import { NextRequest, NextResponse } from "next/server";
import { revalidateTag } from "next/cache";

export async function GET(request: NextRequest) {
  const tag = request.nextUrl.searchParams.get("tag");
  revalidateTag(tag);
  return NextResponse.json({ revalidated: true, now: Date.now() });
}

錯誤處理和重新驗證​

如果在重新驗證數據時出現錯誤，將繼續從緩存中提供最後成功生成的數據。在下一個後續的請求中，Next.js 將重試重新驗證數據。
