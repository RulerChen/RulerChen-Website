## Problem Description

在這個 project 中，我們需要實現一個 key-value store 的 copy-on-write 的 trie，總共有 4 個 Task

### Copy-On-Write Trie

修改 `trie.h` 跟 `trie.cpp` 來實現一個 copy-on-write trie，這個 trie 會有 `GET`、`PUT`、`DELETE` 三個操作。

### Concurrent Key-Value Store

將原本 single-thread 的 trie 改成 concurrent 的版本，一樣有 `GET`、`PUT`、`DELETE` 三個操作。

### Debugging

練習使用 debugger 來 debug，可以參考 `trie_debug_test.cpp`。

### SQL String Functions

這部份是要為 bustub 完成 `lower` 和 `upper` 兩個 SQL 的 function。

## Environment Setup

我使用 WSL (Ubuntu 22.04) + VSCode 來開發這個作業，首先是安裝 VScode 套件 C/C++ Extension Pack、clangd。

接著將專案 clone 下來，並且回退到作業年份的 commit。

安裝 package :

```bash
sudo build_support/packages.sh

mkdir build
cd build
cmake ..
make
```

在安裝完了之後，我們還需要安裝 clangd (通常 vscode 會提示安裝)，接著我們可以設定 formatter，這樣就可以在 save 的時候自動格式化。

```json title=".vscode/settings.json"
{
  "C_Cpp.clang_format_path": "/usr/bin/clang-format-14",
  "C_Cpp.formatting": "clangFormat",
  "C_Cpp.clang_format_style": "file",
  "editor.formatOnSave": true
}
```
